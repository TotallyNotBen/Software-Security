Software-Security Journal

Artemis Financial was the client, a company seeking to modernize its operations by improving the security of its web-based software application, which uses a RESTful API. The primary issue the company wanted addressed was the presence of security vulnerabilities within its system that could expose sensitive financial data to external threats. My role was to assess these vulnerabilities and refactor the code to implement stronger security measures, including encryption, secure communication, and dependency management. This involved upgrading outdated libraries, implementing HTTPS, generating secure certificates, and adding checksum verification using a cryptographic hash algorithm.

One area I performed well in was identifying and addressing vulnerabilities through both automated and manual analysis. Using tools like OWASP Dependency-Check allowed me to detect outdated and vulnerable dependencies, while manual code review helped identify logical and configuration-based security weaknesses. Coding securely is important because it protects data integrity, confidentiality, and system availability. Strong software security adds value to a company by reducing the risk of data breaches, maintaining customer trust, and preventing financial and reputational damage.

One of the more challenging aspects of the vulnerability assessment was interpreting the dependency-check report and determining which vulnerabilities were critical and actionable. However, this process was also helpful because it improved my understanding of how third-party libraries can introduce risks into an application. To increase layers of security, I implemented HTTPS using a self-signed certificate, added SHA-256 hashing for checksum verification, and upgraded dependencies to more secure versions. In the future, I would continue to use tools like OWASP Dependency-Check, static analysis tools, and vulnerability databases such as CVE and NVD to assess risks and select appropriate mitigation strategies.

To ensure the application remained functional and secure, I tested the refactored code by running the application locally, verifying HTTPS connections, and confirming that the checksum endpoint produced correct results. After refactoring, I reran dependency-check to ensure no new vulnerabilities were introduced. I also validated that the application compiled and executed without errors, which confirmed that security enhancements did not break existing functionality.

Throughout this project, I used several valuable resources and tools, including Java Keytool for certificate generation, OWASP Dependency-Check for vulnerability scanning, and Spring Boot for secure application configuration. Secure coding practices such as input validation, encryption, and dependency management were essential in completing this work and will be useful in future projects.

If presenting this work to future employers, I would highlight my ability to identify and remediate security vulnerabilities, implement secure communication protocols, and use industry-standard tools to improve application security. This assignment demonstrates my understanding of secure software development practices and my ability to apply them in a real-world scenario.
